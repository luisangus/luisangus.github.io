What is an API?

API stands for Application Program Interface and it's a set of instructions, methods or routines that work together for building software applications.


In English pleaseeee !!!

Ok, ok, In other words, the API is the messenger that takes requests and tells the system what you want and it comes back with a response. A very basic example would be a person at a restaurant that wants to order some dinner. In this case the person wants to order something off the menu, the messenger is the waiter/(API) that will communicate with the kitchen and let them know your order. The kitchen will prepare the food and when its ready, the waiter/(API) will return back to you with your response/food.

Let's get started

From the PetFinder's website: "The Petfinder API gives developers access to access Petfinder's database of over 300,000 adoptable pets and 11,000 animal welfare organizations (AWO). In addition to searching for adoptable pets on the Petfinder.com web site, you can use the API to create your own dynamic pet web sites or applications, using the same data we use on Petfinder.com"


The link to the PetFinder API is here


In order to take advantage of this excellent API we are going to need an API key and secret. These are given by PetFinder for free, all you need to do is create an account with them.

Click here to register


Once you have registered, you will get an API key and a API Secret. The “key” identifies you every time you make a request through the API and can be shown and transmitted unecrypted. On the other hand the “secret” should be kept a secret and never be shared or displayed, this is used to sign requests that require authentication.


jQuery

Petfinder doesn’t support CORS for Ajax, so you can use this API using proxies or using jQuery. I decided to use jQuery just because at the time it seemed easier for me to grasp the concepts. But what is JQuery you may ask? jQuery is simply a super useful Javascript library designed to simplify and make it easier to write code in our websites.




Examples

Example 1 - breed.list

This API has several method or functions that you can use to search through their database depending on the information you are looking for. In this guide I’ll walk you through most of these methods.
We’ll start with one of the most simple methods we can use with this API and then we'll move into the more difficult ones.
The URL is always going to start with: "http://api.petfinder.com"/


breed.list – This command returns a list of breeds for a particular animal



As you can see, the key and the animal inputs are required and the rest are optional. The format is defaulted to XML but we’ll be using JSON, so we’ll include that in our request. Since we are specifying JSON format we are required to add the callback parameter “callback=?”

Copy the following code and follow along with us. In this example we specify the animal to be a dog and added it to your input parameters, but you can replace that with cat, bird, horse etc.

In your html file all you need is a button and a "div" element to display the response. You can use this line of code 

<input type="submit" id="PetFinderSubmitButton">
<div id="API_response">DOG BREED LIST:</div>

In your javascript file enter this code: 


document.addEventListener('DOMContentLoaded', submitbutton);
  function submitbutton(){
    document.getElementById('PetFinderSubmitButton').addEventListener('click', function(event) {
        displayAPIResponse = document.getElementById('API_response');
          $.getJSON('http://api.petfinder.com/breed.list?key=dcad1eb2f400f451c1fc3b5e4332174d&animal=dog&format=json&callback=?', function(BreedResults) {
            var BreedResults1 = BreedResults["petfinder"]["breeds"]["breed"];
                for (var key in BreedResults1) {
                if (BreedResults1.hasOwnProperty(key)) {
                var newElement = document.createElement('div');
                newElement.innerHTML = key + ": " + JSON.stringify(BreedResults1[key]["$t"]);
                displayAPIResponse.appendChild(newElement);
                }
            }
          });
        event.preventDefault();
      }
    )
  }

The result back is the raw JSON string/object, but the code provided will turn it into a list which will make it look better. In the next examples I’ll show you a function I created to make the result back more easily readable.

Example 2 - pet.getRandom

In this next example we’ll be using a fun method called “pet.getRandom”. This method returns a record for a randomly selected pet. It can be 100% random and you can let PetFinder give you a random response or you can specify parameters to give you a random pet that matches your selected parameters.



The first thing that is required as usual is the “key” and everything else is optional. In this example we'll be getting information regarding a random "Dog", whose breed is "Pug" and we want a "Full" description on the pet. And since we want to use JSON, we need to add the format and callback parameters.

Copy the following code and follow along with us.

In your html file all you need is a button and a "div" element to display the response. You can use this line of code 

<input type="submit" id="PetFinderSubmitButton2">
<div id="API_response2">Information about a random dog:</div>

Example 3 - pet.find

The following example is probably my favorite, the “pet.find” method. This method searches for pets that matches the parameters you select and returns a collection or a list of the pets that match your search. The number of records returned allowed in this request vary depending on the type of developer key PetFinder has allowed you to use. You can set the number of records returned by specifying the “count” per query. For this example we’ll set count = 3



The only 2 parameters that are required are the key and the location in form of a zip or city and state. In this example we'll be searching for all the "Dogs", whose breed is "Pug", the sex is "Male" and are located around "98012" zip code and we only want to get a maximum of 3 results ("count=3" in the URL). And since we want to use JSON, we need to add the format and callback parameters.

Copy the following code and follow along with us.

In your html file all you need is a button and a "ul" element to display the response. You can use this line of code. Notice that we are now using "ul" instead of a div element. 

<input type="submit" id="PetFinderSubmitButton3">
<ul id="API_response3 class="sample">All returned searches:</ul>

In your javascript file enter this code. We introduce a new function for making the response into "list elements", this is to make the response more readable. We'll be using this function for Examples 3, 4, 5 and 6, please add it to those examples as well. 


var body;
function scan(obj, $target)
{
	var k;
        for (k in obj){
            if (obj.hasOwnProperty(k)){
                if (JSON.stringify(obj[k]["$t"]) != undefined) {
                    var li = $('<ul>').appendTo($target); 
                    li.text(k + ": ");
                    var innerList = $('<ul>').appendTo(li);
                    innerList.text(JSON.stringify(obj[k]["$t"]))
                } else {
                    var li = $('<ul>').appendTo($target);
                    li.text(k);
                    scan(obj[k], li);
                }
            } else {
                alert(k);
            }
        }
    return body;
};

        displayAPIResponse3 = document.getElementById('API_response3');

          $.getJSON('http://api.petfinder.com/pet.getfind?key=dcad1eb2f400f451c1fc3b5e4332174d&animal=dog&breed=pug&sex=M&location=98012&count=3&format=json&output=full&callback=?', function(Result) {
		  var Result1 = Result["petfinder"]["pets"];
          body = "";
          scan(Result1, $('.sample'));
          });
        event.preventDefault();
      }
    )
  }

So lets say you want to know which pets are available for adoption are your nearest shelter you just have to input shelter ID and the status as “A” for Adoptable and it will return all the animals in that shelter that are waiting to be adopted. You can also pick to give you a ID of the pet or the full description and that can be specified using the output parameter as “full”.
